# 今天吃什么 - 转盘决策应用

## 项目概述

一个帮助用户决定"今天吃什么"的转盘决策网页应用，通过转盘随机抽选菜品，并提供场景化过滤和个性化偏好功能。

## 技术栈

- **前端**：HTML5 + CSS3 + JavaScript (ES6+)
- **存储**：浏览器 localStorage
- **动画**：Canvas API + requestAnimationFrame
- **部署**：纯静态页面，单 HTML 文件

## 核心功能

### 第一层：MVP（基础抽选）
- 预置 20-30 道常见家常菜/外卖品类
- Canvas 转盘动画抽选
- 添加/删除自定义菜品
- 标签分类：面条类、米饭类、火锅类、小吃类等
- localStorage 持久化

### 第二层：场景化过滤
- 场景切换：早餐 / 午餐 / 晚餐 / 夜宵 / 不限
- 每个场景独立菜品池
- 快速过滤：不想吃辣/不想吃面等

### 第三层：权重与偏好
- 常吃菜品权重提升
- 近期吃过的自动降权
- 踩雷标记 + 避开踩雷选项

## 数据结构

```javascript
{

```

## 开发指南

### 本地运行
直接用浏览器打开 `index.html` 即可，无需服务器。

### 代码规范
- HTML：语义化标签，中文注释
- CSS：BEM 命名规范，响应式设计
- JavaScript：ES6+ 语法，函数式编程风格
- 注释：复杂逻辑必须添加注释，解释"为什么"而不是"做了什么"

### 命名规范
- 变量/函数名：见名知意，拒绝 `a`, `b`, `foo` 等无意义命名
- CSS 类名：BEM 规范，如 `.wheel__button--active`
- 常量：大写下划线，如 `MAX_DISH_COUNT`

## 注意事项

1. **数据持久化**：所有数据存储在 localStorage，清除浏览器数据会丢失
2. **性能优化**：转盘动画使用 requestAnimationFrame，避免卡顿
3. **响应式**：支持手机和电脑，使用 Flexbox + 媒体查询
4. **浏览器兼容**：现代浏览器（Chrome、Firefox、Safari、Edge）
